## Attack Surface

This tool has a minimal attack surface by design:

- **No network access.** The application makes zero network connections. No telemetry, no update checks, no remote endpoints. Everything runs locally.
- **No kernel access.** This is a userspace application. It uses standard Windows APIs (`SetProcessAffinityMask`, `GetLogicalProcessorInformationEx`, PDH counters) that do not require elevated privileges or driver signing.
- **No credentials.** The application does not store, transmit, or process any credentials, tokens, or authentication material.
- **Standard user only.** The application runs under standard user permissions. It cannot modify system processes — access denied errors for protected processes are expected and handled gracefully.
- **Local config only.** Configuration is a JSON file at `%APPDATA%\X3DCCDOptimizer\config.json`. It contains process names and UI preferences. No sensitive data.
- **No IPC.** No named pipes, no shared memory, no sockets, no COM objects exposed.

## What the Application Does Access

For transparency, here is what the application does interact with:

- **Process enumeration** — reads the list of running processes via `System.Diagnostics.Process.GetProcesses()`
- **Process affinity** (Optimize mode only) — calls `SetProcessAffinityMask` via P/Invoke to change which CPU cores a process can run on. This is a standard, reversible Windows API call. Monitor mode never calls this API.
- **CPU topology** — queries CPU cache topology via `GetLogicalProcessorInformationEx` and WMI `Win32_CacheMemory`
- **Performance counters** — reads per-core CPU load and frequency via PDH (Performance Data Helper) counters
- **Foreground window** — calls `GetForegroundWindow` to identify which application is in the foreground
- **File system** — reads/writes its own config file and log files in `%APPDATA%\X3DCCDOptimizer\`

All of these are standard, documented Windows APIs available to any userspace application.

## Reporting a Vulnerability

If you find a security issue, please report it responsibly:

1. **Do not open a public issue.** Instead, email the maintainer or use GitHub's private vulnerability reporting feature on the repository.
2. Include a description of the vulnerability and steps to reproduce.
3. Allow reasonable time for a fix before public disclosure.

Even though the attack surface is minimal, security reports are taken seriously. If you find something, we want to know.

## Scope

The following are **not** security vulnerabilities in this project:

- **Anticheat detection.** Some anticheat systems may flag affinity changes. This is expected behaviour, not a vulnerability. Use Monitor mode if this is a concern.
- **Affinity changes affecting game performance.** The tool modifies process affinity as designed. If a specific game behaves poorly, that's a compatibility issue.
- **Access denied errors.** The tool intentionally skips system processes it cannot modify. This is correct behaviour, not a failure.
- **Config file tampering.** The config file is in the user's own `%APPDATA%` directory. If an attacker has write access there, they already have far more dangerous options than editing this tool's config.

## Monitor Mode as a Safety Default

The application starts in Monitor mode by default. In this mode, it never calls `SetProcessAffinityMask` or any other API that modifies process state. It only reads system information and displays it. This makes Monitor mode safe to run on any system without concern about unintended side effects.

Optimize mode must be explicitly enabled by the user and requires confirmed V-Cache detection to activate.
