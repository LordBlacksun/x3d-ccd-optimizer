# Contributing to X3D CCD Inspector

Thanks for your interest in contributing. This project is open source under GPL v2 and actively welcomes community contributions.

## How This Project Is Built

Architecture, design decisions, and project direction come from LordBlacksun. Implementation is generated by [Claude Code](https://claude.ai/claude-code) under human supervision -- every change is reviewed, tested, and approved before commit.

Community code reviews are especially valuable. A second pair of human eyes catching things the review process missed is exactly the kind of contribution that makes open source work. If you spot something that looks off -- a logic error, an edge case, a pattern that doesn't make sense -- please open an issue or PR.

## Getting Started

### Prerequisites

- Windows 10 21H2+ or Windows 11
- .NET 8 SDK
- An AMD dual-CCD Ryzen processor (for runtime testing -- not required for code contributions)

### Building

```bash
git clone https://github.com/LordBlacksun/x3d-ccd-optimizer.git
cd x3d-ccd-optimizer
dotnet build
dotnet test
```

### Running

```bash
dotnet run --project src/X3DCcdInspector
```

## How to Contribute

### Reporting Bugs

Use the [bug report template](https://github.com/LordBlacksun/x3d-ccd-optimizer/issues/new?template=bug_report.md). Include:
- Your CPU model (e.g., Ryzen 9 7950X3D)
- Windows version
- Steps to reproduce
- Log file from `%APPDATA%\X3DCCDInspector\logs\`

### Code Contributions

1. Fork the repository
2. Create a feature branch from `develop`: `git checkout -b feature/your-feature develop`
3. Make your changes
4. Ensure `dotnet build` succeeds with no warnings
5. Run `dotnet test` to verify all tests pass
6. Test on hardware if possible
7. Submit a pull request targeting `develop`

### Code Reviews

Areas where review is particularly valuable:

- **P/Invoke correctness** -- signatures, struct layouts, marshalling
- **Thread safety** -- lock coverage, Dispatcher usage, disposal races
- **Edge cases** -- unusual CPU topologies, process states, permission scenarios
- **Registry operations** -- AMD driver per-app profile reads/writes
- **Resource cleanup** -- native handles, PDH queries, timers

## Branch Strategy

- `master` -- stable releases only
- `develop` -- active development, PRs target this branch
- `feature/*` -- feature branches off develop

## Code Guidelines

- Target .NET 8 (`net8.0-windows`), WPF + WinForms (for NotifyIcon)
- Enable nullable reference types
- All P/Invoke calls must have `SetLastError = true` and proper error checking
- All process operations wrapped in try/catch -- never crash on a single process failure
- Protected process list enforced before any affinity modification
- Log significant actions at INFO, failures at WARNING or ERROR
- Use `IDisposable` for anything holding native handles or timers
- Use events for inter-module communication
- Case-insensitive process name matching everywhere
- Config load/save wrapped in try/catch -- never crash on corrupted config
- MVVM for WPF: Views bind to ViewModels, no logic in code-behind except window management
- All engine events marshalled to UI via `Dispatcher.BeginInvoke` (not `Invoke`)

## Project Structure

```
src/X3DCcdInspector/
├── Core/       # Engine (CcdMapper, PerformanceMonitor, ProcessWatcher, GameDetector,
│               #   GpuMonitor, AffinityManager, VCacheDriverManager, SystemStateMonitor)
├── ViewModels/ # MVVM ViewModels (Main, CcdPanel, CoreTile, ActivityLog, ProcessRouter,
│               #   Overlay, SystemStatus, ActiveGame, GameLibrary, Settings)
├── Views/      # WPF XAML (DashboardWindow, CcdPanel, CoreTile, OverlayWindow, Settings, About)
├── Themes/     # Resource dictionaries (DarkTheme, Typography, Controls)
├── Converters/ # WPF value converters
├── Tray/       # System tray (TrayIconManager, IconGenerator)
├── Config/     # JSON configuration (AppConfig)
├── Logging/    # Serilog setup
├── Native/     # P/Invoke (Kernel32, User32, Pdh, Structs)
├── Models/     # Data models (CpuTopology, CoreSnapshot, AffinityEvent, SystemState, ScannedGame)
└── App.xaml    # WPF Application entry point
```
